# 总体开发流程与部署步骤 (Workflow & Deployment)

本项目定位为**极其轻量的朋友聚会在线桌游平台**。主打：免注册、无数据库、无复杂特效，通过链接或房间号一键加入。

## 一、核心用户流程体验 (User Flow)
1. **统一主页 (Game List)**: 作为平台入口，展示目前支持的桌游列表卡片。
2. **游戏落地页大厅 (Game Lobby)**: 点击游戏后进入。
   - 提供两个选项：“创建房间 (Create)” / “加入房间 (Join)”。
   - 填写玩家昵称 (Player Name)。
3. **房间专属大厅 (Room)**: 房间建成后，其他玩家通过房间号进入此界面。
   - 展示当前的 Player 列表。
   - 房主负责设定房间参数 (Player amount, Game settings)。
   - 提供一份简要的规则预览面板 (See Game Rules)。
4. **游戏场景 (Game Board)**: 房主点击“开始游戏”，全体跳转/切换为桌游下棋/出牌界面。

---

## 二、零成本全免费部署方案
我们放弃一切需要付费或复杂运维的云服务器（AWS/阿里云）与数据库，选择拥抱免费的 Serverless / PaaS 方案，做到“一天内部署上线”：
- **前端部署**：Vercel 或者 Netlify。完全免费的静态应用托管，推送到 GitHub 即可自动打包更新，带有极速 CDN。
- **后端 WebSocket 服务**：选择 Render 的免费层 (Web Service) 或是 Railway / Fly.io 免费节点。
  - *注：免费服务通常在无人访问 15 分钟后宕机睡眠，下次访问会有数十秒的唤醒冷启动，但作为一款“聚会时拉人玩”的游戏，这种“白嫖”策略完全没有任何体验问题，提前打开网页亮个机即可。*

## 三、总体实现分期步骤 (Milestones)

### 阶段一：打通极简 MVP (Minimum Viable Product)
- [ ] 初始化全栈项目（前端 React + 后端 Node.js）。
- [ ] 将“Hello World”推送发布至 Vercel 和 Render 上，确保公网可以访问到网页，并且 WebSocket 能跨域连接成功。
- [ ] 实现基础的房间路由：创建一个随机 4 位数 Room Code，能让朋友的手机输入房间号并成功加入。

### 阶段二：平台骨架与通用设施搭建
- [ ] 制作主页 Game List 和统一的房间等待大厅 UI。
- [ ] 完善从输入名字、加入房间、调整人数设置到集体收到“Game Start”指令的流程闭环。

### 阶段三：填充第一个桌面游戏逻辑
- [ ] 开发选定的首发桌游：在前端构建干净的局内操作 UI。
- [ ] 在后端编写并维护这个游戏的逻辑状态机器（比如该谁的回合了、谁赢了）。

### 阶段四：测试优化
- [ ] 确保移动端全覆盖：针对手机浏览器的横屏/竖屏排版做自适应优化。
- [ ] 处理弱网环境下的刷新与断路重连体验（让朋友刷新网页不会丢失这盘棋）。
